XFS Interface Specification

XFS Interface is a command line external interface to access the NITCBase filesystem from the host (UNIX) system. The filesystem is simulated on a binary file. This interface aims to provide the functionality of transferring relations from/to the host system to/from the disk and can be helpful in debugging the system during implementation. .There are three types of commands that can be given as input to the XFS interface:

  1. External File System Commands
  2. Data Definition Language Commands
  3. Data Manipulation Language Commands

External File System Commands

The External File System commands are used to format the disk, dump data structures like Block allocation map,Relation catalog and Attribute catalog, load/remove relations, list relations and copy specified block of a relation on the XFS disk to a UNIX file. The following are the External File System commands that can be queried into NITCbase :

Format Disk

Syntax :

fdisk

Description:

It is used to create the disk (”disk”) or to format the disk if already created. On a newly created/formatted disk default entries for the Block allocation map, Relation Catalog and Attribute Catalog are initialized according to the specification for the physical layer of NITCBase.

Import Relation

Syntax :

import filename

Description:

The command import is used to load relations from the UNIX filesystem to the XFS disk.The argument filename specifies the name of the CSV file which contains the contents of the relation to be uploaded. TheFilenames should not contain whitespaces or any special characters except '-' or '_'. The command checks the size of the relation in the csv file, allocates the required number of blocks for the relation, updates the Block allocation map, Relation catalog and Attribute catalog.

File Format:

The contents of the relation must be in a file and must follow CSV (comma separated values) format.The first line of the CSV file must contain the names of the attributes of the relation. The data types of the attributes in the first line are inferred from the values of the corresponding attributes on the second line of the CSV file. The name of the csv file must be the same as the relation to be imported to the disk. i.e. the CSV file name should be in the format "relname.csv". All files to be imported should be stored in the path $HOME/NITCBase/Files . If a relation with the same name as that of the CSV file already exists, then the import will not happen. First 15 characters of name of file will be taken as relation name. Also the first 15 characters of attribute values will only be taken. The csv file should not contain any null values. The number of attribute values in each row should match the number of attributes.(ie the number of attribute values in first row).

Example:

Consider the contents of the file 'Students.csv'.

No, Name, Cgpa
3,Sunny,8.2
5,Sania,6.0
7,Ralph,7.5

"import sample.csv" command will import relation 'Students' into the disk.

The first line in the csv file represents the list of attributes in the relation which in this case are No, Name, Cgpa. The datatypes of the attributes are determined from the values of the attributes in the second line. An attribute can be an integer,float or a string. In this example the datatypes will be integer, string, float respectively.

Export Relation

Syntax :

export tablename filename

Description:

This command is used to export a relation from XFS disk to UNIX file system.The contents of the relation are written to the specified file given by filename, in the CSV format specified above, in the path $HOME/NITCBase/Files .

Example:
The following command will export relation "Students" to a csv file named "Marks". 
export Students Marks.csv

List Relation Names

Syntax :

ls

Description:

This command is used to list the names of all relations in the xfs disk.

Dump Block allocation map

Syntax :

dump bmap

Description:

This will create a file named bmap.txt and dump the contents of the block allocation map into the file.

Dump Relation Catalog

Syntax :

dump relcat

Description:

This command is used to copy the contents of relation catalog to an external file named relationcatalog.txt.

Dump Attribute Catalog

Syntax :

dump attrcat

Description:

This command is used to copy the contents of attribute catalog to an external file named attributecatalog.txt.

Exit

Syntax :

exit

Description:

This command is used to exit the XFS interface.

DDL Commands

The Data Definition Language commands are used to define the database schema. They are used to create and modify the structure of relations in the database. The following are the DDL commands that can be queried into NITCbase :

CREATE TABLE

Syntax :

CREATE TABLE tablename(attr1_name attr1_type ,attr2_name attr2_type....);

Description:

This command is used to create a relation of the given name, with given attribute names and types.The type of an attribute can be INT, STR or FLOAT.

Note:

In NITCBase, the maximum size of an attribute is 16 bytes. Since relation names and attribute names are attributes themselves in the catalog structures, the table name and attribute names in the queries must only have a maximum of 15 characters. If the length is greater than 16, only the first 15 characters will be taken.A relation cannot be named as "temp", since it is used for internal operations.

Example:
CREATE TABLE sample (Rollno INT,Name STR,CGPA FLOAT);

DROP TABLE

Syntax :

DROP TABLE tablename ;

Description:

This command is used to delete the relation of the given name. It deletes all the record and index blocks corresponding to the relations, and also delete the entries corresponding to the relation in the relation catalog and attribute catalog.The entries corresponding to the deleted blocks in the block allocation map are also reset.

Example:
DROP TABLE sample ;

OPEN TABLE

Syntax :

OPEN TABLE tablename ;

Description:

This command is used to open the relation specified in cache/OpenRelTable.

Example:
OPEN TABLE sample;

CLOSE TABLE

Syntax :

CLOSE TABLE tablename ;

Description:

This command is used to close the relation specified, in cache/OpenRelTable.

Example:
CLOSE TABLE sample;

CREATE INDEX

Syntax :

CREATE INDEX ON tablename.attributename ;

Description:

This command is used to create an index on a given attribute of a relation.Before executing this query,the relation must be opened first using the OPEN TABLE query.

Example:
CREATE INDEX ON sample.Rollno ;

DROP INDEX

Syntax :

DROP INDEX ON tablename.attributename ;

Description:

This command is used to drop/delete the B+ tree indexing on the given attribute of the given relation.Before executing this query,the relation must be opened first using the OPEN TABLE query.

Example:
DROP INDEX ON sample.Rollno ;

ALTER TABLE RENAME

Syntax :

ALTER TABLE RENAME tablename TO new_tablename ;

Description:

This command is used to rename an existing relation to a given new name.

Example:
ALTER TABLE RENAME sample TO Students;

ALTER TABLE RENAME COLUMN

Syntax :

ALTER TABLE RENAME tablename COLUMN column_name TO new_column_name ;

Description:

This command is used to rename an attribute of an existing relation.

Example:
ALTER TABLE RENAME sample COLUMN CGPA TO SGPA;

DML Commands

The Data Manipulation Language commands are used to manipulate the data stored in the relations of the database. The following are the DML commands that can be queried into NITCbase :

INSERT INTO TABLE VALUES

Syntax :

INSERT INTO tablename VALUES ( value1,value2,value3,... );

Description:

This command is used to insert a single record into the given relation. The attribute values of the record are to be given as comma separated values. The order of the values must be the same as the order of attributes of the relation. For example, if a relation Students was created

Example:

Given below are the records of the relation Students.

Rollno Name CGPA
1 Anu 9.01
4 Cody 7

Suppose that we need to insert a new record "2, Amy, 9.5" into the relation Students.

The query will look as follows:
INSERT INTO Students VALUES (2,Amy,9.5);

The relation Students now looks like:

Rollno Name CGPA
1 Anu 9.01
4 Cody 7
2 Amy 9.5

INSERT INTO TABLE FROM FILE

Syntax :

INSERT INTO tablename VALUES FROM filename;

Description:

This command is used to insert multiple records into the relation from a CSV file containing the values for the corresponding attributes. The order of values in the CSV file must be the same as the attributes of the relation. Each line in the CSV file corresponds to a record to be inserted in to the relation.The argument filename specifies the name of the CSV file which contains the contents of the relation to be uploaded. Filenames should not contain whitespaces or any special characters except - or _. The CSV file should not contain any null values. The number of attribute values in each row should match the number of attributes.(ie the number of attribute values in first row). The CSV file from which the values are to be inserted, must be stored in the path $HOME/NITCBase/Files .

Example:

Here is an example of how a csv file containing records for insertion looks like:

students.csv

3,Sunny,8

5,Sania,6

7,Ralph,7.5


The query to insert all records from students.csv into the relation Students is :
INSERT INTO Students VALUES FROM students.csv;

SELECT * FROM TABLE

Syntax :

SELECT * FROM source_relation INTO target_relation ;

Description:

This command creates a new target relation with the same attributes as that of source relation,and inserts into it all records from the source relation

Example:
SELECT * FROM Students INTO Target_Students;

SELECT Attrlist FROM TABLE

Syntax :

SELECT Attribute1,Attribute2,....FROM source_relation INTO target_relation ;

Description:

This command creates a new target relation with the attributes specified in Attrlist,and inserts all records(only the values corresponding to the specified attributes) of the source relation, into the newly created target relation.

Example:
SELECT Name,CGPA FROM Students INTO Target_Students;

SELECT * FROM TABLE WHERE

Syntax :

SELECT * FROM source_relation INTO target_relation WHERE attrname OP value;

Description:

This command is used to retrieve all records of a given source relation, and insert them into a target relation, based on the the given condition. All records in the source relation that satisfy the condition, will be inserted into the newly created target relation.

Example:
SELECT * FROM Students INTO Target_Students WHERE CGPA > 8;
Note:

Here, OP can only take a value from the set {=,>,<.>=,<=,!=}.

SELECT Attrlist FROM TABLE WHERE

Syntax :

SELECT Attribute1,Attribute2,....FROM source_relation INTO target_relation ;

Description:

This command creates a new target relation with the attributes specified in Attrlist ,and inserts those records(only the values corresponding to the attributes specified in the Attrlist) from the source relation which satisfy the given condition.

Example:
SELECT Name,CGPA FROM Students INTO Target_Students WHERE CGPA>8;

SELECT * FROM JOIN WHERE

Syntax :

SELECT * FROM source_relation1 JOIN source_relation2 INTO target_relation WHERE source_relation1.attribute1 = source_relation2.attribute2 ;

Description:

This command creates a new target relation with attributes constituting from both the source relations(excluding specified attribute from 2nd src relation).It inserts the records obtained by equi-join of both the source relations(an attribute from each relation specified in arguments are used for equi-join) into the target relation.

Note that attribute1 should belong to source_relation1 and attribute2 should belong to source_relation2.

Example:

Given below are the records of the relation Student1.

Rollno Name Batch
1 Anu A
2 Cody B
3 Amy B

Given below are the records of the relation Student2.

Rollno Marks
1 98
2 80
3 97
4 67

An example for a join query is:

SELECT * FROM Students1 JOIN Students2 INTO Students WHERE Students1.Rollno = Students2.Rollno;

Equi-join on these two relations based on the attribute Rollno would result in the following target relation,Students.

Rollno Name Batch Marks
1 Anu A 98
2 Cody B 80
3 Amy B 97

SELECT Attrlist FROM JOIN WHERE

Syntax :

SELECT Attribute1,Attribute2,.. FROM source_relation1 JOIN source_relation2 INTO target_relation WHERE source_relation1.attribute1 = source_relation2.attribute2 ;

Description:

This command creates a new target relation with attributes given in Attrlist.It inserts the records(only the values of the specified attributes in Attrlist obtained by equi-join of both the source relations(an attribute from each relation specified in arguments are used for equi-join) into the target relation.

Note that attribute1 should belong to source_relation1 and attribute2 should belong to source_relation2.

Example:

Example to be given

Given below are the records of the relation Student1.

Rollno Name Batch
1 Anu A
2 Cody B
3 Amy B

Given below are the records of the relation Student2.

Rollno Marks
1 98
2 80
3 97
4 67

An example for a join query is:

SELECT Rollno,Name,Marks FROM Students1 JOIN Students2 INTO Students WHERE Students1.Rollno = Students2.Rollno;

Equi-join on these two relations based on the attribute Rollno would result in the following target relation,Students.

Rollno Name Marks
1 Anu 98
2 Cody 80
3 Amy 97
