Indexing in NITCBase

NITCBase uses B+ Tree for indexing an attribute in a relation. To implement B+ Tree layer you must require a thorough understanding of Physical layer, Buffer layer and Cache layer. As you might already know, we use indexing to reduce disk access and to implement search queries faster. Click here to read more on B+ Trees.

Understanding Indexing

On creating a database of your own, it is essential that we have access to the data stored without much time delay. Hence we use indexing. Once a database is created and stored in a disk we cannot always search for a particular entry when a need arises by starting from the beginning. Creating an index enables us to access the required entry with very few disk access. Disk access is a high-latency operation, which is not suitable for a very large database. Hence we must find a way to get access to each entry without wasting time on disk access. This paves the way for B Trees and B+ Trees.
